# Rename Report: `loadout` → `loadouts`

**Premise:** The _tool_ is called `loadouts`. A _bundle_ managed by it is called a `loadout`. This clarifies the singular/plural distinction throughout the codebase, docs, and user-visible paths.

---

## Decision Points (Resolve Before Starting)

### D1: Project config directory name

The most impactful decision. Currently `.loadout/`.

| Option | Result | Tradeoff |
|--------|--------|----------|
| `.loadouts/` | `.loadouts/loadouts/base.yaml` | Consistent with tool name; redundant subdir |
| Keep `.loadout/` | `.loadout/loadouts/base.yaml` | No redundancy, shorter; inconsistent with tool name |

**Recommendation:** `.loadouts/` — consistency wins. The "redundancy" is actually clarity: the outer dir is the tool config root, the inner `loadouts/` dir is where bundles live.

### D2: Root config filename

Currently `loadout.yaml` inside the project dir.

| Option | Example path | Note |
|--------|-------------|------|
| `loadouts.yaml` | `.loadouts/loadouts.yaml` | Consistent with tool name |
| `config.yaml` | `.loadouts/config.yaml` | Clean but breaks the naming pattern |
| Keep `loadout.yaml` | `.loadouts/loadout.yaml` | One less change; reads well as "the loadout config" |

**Recommendation:** Keep `loadout.yaml`. It names the _thing being configured_ (a loadout), not the tool. `.loadouts/loadout.yaml` reads as "the loadout config, inside the loadouts tool dir" — perfectly coherent.

### D3: Migration command

At 0.1.10, real users may have `.loadout/` dirs, `~/.config/loadout`, and `.gitignore` markers.

**Recommendation:** Add a `loadouts migrate` command that renames `.loadout/` → `.loadouts/`, moves `~/.config/loadout` → `~/.config/loadouts`, and updates `.gitignore` markers.

### 3. Filesystem Paths

These constants drive all path generation. Changing them cascades to all disk I/O.

**`src/core/discovery.ts`**
```ts
// Before
const LOADOUT_DIR = ".loadout";
export function getGlobalConfigPath() {
  return path.join(os.homedir(), ".config", "loadout");
}

// After
const LOADOUT_DIR = ".loadouts";
export function getGlobalConfigPath() {
  return path.join(os.homedir(), ".config", "loadouts");
}
```

**`src/core/config.ts`** — `loadout.yaml` filename unchanged per D2; `LOADOUTS_DIR = "loadouts"` already correct.

**Impact:** Every `path.join(..., ".loadout")` throughout the codebase resolves from this constant. No individual site changes needed beyond the constant itself.

### 5. Generated File Content

**`src/core/import-discovery.ts`** — detection heuristic must match the new string:

**`src/core/fallback.ts`** — shell script template embedded as a string. Contains:
- `[ -f ".loadout/.state.json" ]` → `.loadouts/.state.json`

- `loadout sync -l` → `loadouts sync -l`

- `for rule in .loadout/rules/*.md` → `.loadouts/rules/*.md`
- `for skilldir in .loadout/skills/*/` → `.loadouts/skills/*/`
- `exec .loadout/sync-fallback.sh` → `.loadouts/sync-fallback.sh`
- Install message: `npm install -g loadout` → `npm install -g loadouts` (or `@evatt/loadouts`)

## What Does NOT Change

To preserve the singular/plural distinction correctly:

| Item | Reason |
|------|--------|
| TypeScript types: `LoadoutDefinition`, `ResolvedLoadout`, `LoadoutRoot` | These name _a single bundle_ — singular is correct |
| Function names: `resolveLoadout()`, `parseLoadoutDefinition()`, `findLoadoutDefinition()` | Operate on a single bundle |
| Variable names: `loadout`, `loadoutName`, `loadoutRoot` in function bodies | Local variables referring to one bundle |
| `LOADOUTS_DIR = "loadouts"` in `config.ts` | Already correct — the subdir storing multiple bundles |
| `lib/loadout-column.ts` — `loadoutName` field | Column showing a single loadout's name |
| `"loadout"` as a column header label in tables | This labels the bundle column — semantically correct |
| `.argument("<names...>", "Loadout names to activate")` | Still refers to bundle names |
| `--loadout <name>` option in `rule` and `skill` commands | Targets a specific bundle |

7. **Bundled assets** (`bundled/loadouts/loadout.yaml` name field)

10. **Migration command** (if doing D3) — rename dirs, update `.gitignore` markers
11. **`npm install`** — regenerates `package-lock.json`
